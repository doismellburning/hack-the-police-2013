Guidance

How to write mobile apps that store sensitive data safely.

Best practises

The field of mobile policing is still evolving, and we are still working on best practises in this area. This will be an ongoing conversation that we look forward to having with you at the hackathon.

Before the hack, this page will show what we consider to be the essentials - the basic requirements for an app that may need to store sensitive and personal data.

Frameworks

We're not going to judge you on this!

We want you to be as creative as possible using the skills that come to you fastest and easiest -- but if you want a suggestion for a framework to work in, read on:

If you're building a mobile app, please consider building it in a device-agnostic fashion - perhaps as a mobile HTML5 app, using frameworks such as jQuery Mobile and/or Cordova.

Storage

Sensitive information has to be handled carefully and kept secure.

We've mocked up a remote storage solution - called RecordStore, which your mobile apps can use to store and retrieve objects.

For the purposes of the hack (fast development, ease of access), the record store has not been secured - nor is it a production application. We've hacked it together to be representative of the functionality you might find in production solutions, and to help you get stuck-in to creating apps as quickly as possible.

For information about how to get an appId and how to use the APIs, read the Hackathon APIs: Record Store documentation.
